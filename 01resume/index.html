todo - iOS App

todo - iOS AppMobile

todo (pronounced “toe doe”) is a point-based, tutoring application that allows students to gain dots (i.e., points) by offering tutoring to other students, as well as spend dots in order to receive tutoring from other students. todo app is intended to be an immediate, easy and affordable way for students to get a private assistance or help on their subject of difficulty, while paying the helper time and effort with a system of points. We named the above mention points as “dots”. Dots will work as an alternative currency or exchange system where the unit of account/value is depending of the person-hour. This dots-bases system value everyone’s contributions equally: one hour equals some service dots credit. The idea is that one person volunteers to work for some time for another person; thus, they are credited with a number of dots which they can redeem for a same time amount dots of service from another volunteer.

I was one of three developers on the team that built the iOS app.

I worked on all sides and functionalities of the project, but mainly on database modeling, implementation and coordination of tutor/student request, response, pairing, dots transactions, wallet, ...

Technologies: Swift, Objective-C, Cocoa Touch, Xcode, Firebase, RESTful, JSON.

Source Code and Documentation on GitHub

GotoPaws - Web App

GotoPaws - Web App Web

GotoPaws purpose is to provide one place for people to find information about shelters in their area, parks, vets, and groomers, as well as pets up for adoption.

The rackspace open cloud platform is used to host the website on a URL provided by namecheap. The rackspace VM utilizes Django 1.8.2 to create the framework for the website and to link URLs to html pages. Each model is made of a header, a small descriptions and a table. The table is real time searchable and sortable in all major fields except for images.

Apiary is used to create an API mockup and documentation. The api documentation is available at here. The database models can be viewed in UML form at here.

Whenever a user goes to a URL, either from a link from an html page or from typing it in, Django uses a URL dispatcher to find and load the appropriate html page as well as give that page database information if needed.

I worked mostly on the MVC design and back-end of the Project, including UML, MySQL database design, models design, a lot fo django stuff, ElasticSearch implementation.

Technologies: Python, Django, UML, MVC, MySQL, RESTful API, Apiary, ElasticSearch, JSON, Bootstrap, HTML/XML, JavaScript, JQuery

Source Code and Documentation on GitHub

PhotoBall - Android App Mobile
